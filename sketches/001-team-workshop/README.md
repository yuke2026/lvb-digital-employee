## Variant: 团队工坊 (Team Workshop)

### 设计立场
对话驱动的团队协作界面。用户通过**自然语言**与各个数字员工沟通，像跟真人同事聊天一样派发任务。

### 关键设计选择
| 维度 | 选择 |
|:---|:---|
| **布局** | 三栏式：左-团队列表 / 中-对话区 / 右-产出物面板 |
| **交互核心** | 聊天界面，以对话替代菜单操作 |
| **团队主管** | 默认选中「团队主管」→ 一句话就能派活给整个团队 |
| **产出物面板** | 右侧实时展示各成员产出的工件（PRD、代码、设计稿、测试报告） |

### 优势与劣势
- ✅ **强项**：上手极快，符合聊天习惯；一句话即可启动复杂流程
- ✅ **强项**：每个数字员工有独立人格和回复风格
- ✅ **强项**：适合快速原型验证——跟PM说完需求就看到PRD框架
- ⚠️ **弱项**：批量操作不如看板直观
- ⚠️ **弱项**：任务状态需要从对话中感知

### 适用场景
- 快速需求验证（「帮我做个登录模块」→ 立即产出）
- 管理者视角——随时找人问进度
