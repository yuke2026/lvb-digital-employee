## 为什么这个架构好

### 百应智星视角（你）

| 好处 | 说明 |
|:---|---:|
| **已有现成的飞书接入** | 百应智星已经对接了飞书自建应用，物流机器人直接复用 |
| **已有数字员工体系** | 只需在员工列表里加一个 📦 物流客服，就整站可用 |
| **复用基础设施** | 同一台服务器、同一套认证、同一个数据库 |
| **无需额外部署** | 代码放 `backend/app/services/logistics/` 即可 |

### 独立商业化视角（客户）

| 好处 | 说明 |
|:---|---:|
| **一键部署脚本** | `curl 脚本 | bash` 就能装好，适合不懂技术的快递站老板 |
| **全独立运行** | 不依赖百应智星，每个快递站自己一个实例 |
| **轻量级** | 一个 FastAPI + SQLite，配置好飞书和极兔Key就能跑 |
| **可白牌** | 换成自己的品牌名「XX物流助手」再出售 |

## 共享核心 (services/logistics/) 的关键组件

```
📦 物流客服数字员工
│
├── 指令解析 ──── 关键词「拦截退回」+ 正则提取运单号(UT\d+)
│                宽松匹配，客服可自由描述
│
├── 极兔API调用 ── 三层签名算法
│   1. 密文 = MD5(明文密码+"jadada236t2").toUpperCase()
│   2. digest = Base64(MD5(客户编号+密文+privateKey))
│   3. body.bizContent = JSON字符串
│
├── 回传回调 ──── 极兔回调 → 查操作记录 → 群内通知结果
│
└── 责任声明 ──── 每条回复末尾附责任声明，明确操作人担责
```
