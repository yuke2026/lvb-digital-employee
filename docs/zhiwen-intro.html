Challenge & Solution

CEO 信息困境

企业管理者每天面临数百条行业信息，靠人工筛选既不现实也不可持续。

😰

传统方式

CEO每天花费 1-2 小时浏览新闻和行业报告，信息源分散在微信公众号、行业媒体、朋友圈，信息过载且无法持续追踪。关键趋势往往在爆发后才被注意到，错失决策窗口。

🧠

智闻方案

AI 驱动的行业情报系统，7×24 小时自动采集多源信息，AI 智能分析生成结构化周报/月报，识别风险与机会，主动推送决策简报。从「人找信息」变成「信息找人」。

01 📡 多源信息采集

系统内置三个权威行业 RSS 源（36氪、雷锋网、iThome），同时支持手动添加任意 RSS Feed。每 6 小时自动轮询，增量采集最新文章，URL 级去重确保不重复。

RSS Feed36氪雷锋网iThome增量同步

02 🔍 智能关键词匹配

每个主题可配置专属关键词和排除词。新采集的文章自动与主题关键词做模糊匹配，匹配成功的文章标记为「已处理」，可同时关联多个新闻源。支持中文分词级别的模糊匹配，准确率高。

关键词过滤排除词多源关联中文分词

03 🤖 AI 深度分析（SWOT + 风险 + 机会）

基于 DeepSeek 大模型，对匹配文章进行结构化 SWOT 分析：优势(S)、劣势(W)、机会(O)、威胁(T)。自动提取 🔴 风险清单（高/中/低三级评估）和 🌟 机会清单（含潜力评估和时间窗口）。分析结果封装为 JSON 结构，保证前端和推送链路的稳定解析。

DeepSeekSWOT风险识别机会发现JSON结构化

04 📊 多周期报告生成

支持 5 种报告周期：日报、周报、月报、季报、年报。使用日历周期（非简单「过去N天」），报告标题自动匹配自然时间标签（如「2026年第23周」「2026年Q2」）。向上兼容调度：配置周报的主题自动也会按时生成月报、季报、年报。1号重叠检查机制避免月报/周报/日报同时生成。

日报周报月报季报年报日历周期

05 📨 多渠道主动推送

报告生成后自动通过飞书 Webhook 卡片推送到群聊。卡片包含摘要、SWOT 概要、风险识别项（带 🔴🟡🟢 等级图标）、机会发现和文章快照链接。支持飞书 OAuth API 和 Webhook 双通道，任一路径失败不影响另一路。

Capabilities

核心优势

普通的信息聚合工具 ≠ 智能情报系统。智闻的每一项能力都围绕「决策效率」设计。

📚

多主题并行追踪

支持创建任意数量的行业主题，每个主题独立配置关键词、新闻源和报告周期，互不干扰。可同时追踪「机器人出海」「AI最新动态」「新能源汽车」等行业。

📅

5级日历周期

日报/周报/月报/季报/年报，使用自然日历范围，自动错开重叠周期。每月1号只生月报不生成日报和周报，避免重复。

🎯

AI SWOT 分析

DeepSeek 驱动的结构化分析引擎，输出优势/劣势/机会/威胁四象限，附带风险清单和商业机会清单，每项都引用源文章支撑。

🔔

主动推送到飞书

报告生成后自动推送到企业飞书群，CEO 无需登录系统即可在聊天中阅读。卡片包含摘要、风险等级标识和源文章链接。

⏰

精细化自动调度

每个主题三个独立开关：数据采集、报告生成、飞书推送。可独立启停，APScheduler 实时生效，无需重启服务。

🔒

数据安全可靠

单机 SQLite 部署，无需外部数据库。每日自动全量备份，保留30天历史。Git 实时推送代码到 GitHub，双重保障。

⚡

一键手动刷新

支持手动触发全量刷新：采集最新 RSS → 关键词匹配 → 生成 AI 报告。修改关键词后可立即重跑，无需等待定时调度。

🔄

空周期自动回溯

当前周期无数据时自动回溯7天查找文章，保证报告不出现空内容。回溯数据在报告中标注「含回顾」标记。

🔐

多组织隔离

基于 org_id 的数据隔离，每个注册用户自动分配独立组织。不同企业的主题、报告、推送配置完全隔离。
