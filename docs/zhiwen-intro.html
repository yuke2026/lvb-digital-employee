Challenge & Solution

CEO 信息困境

企业管理者每天面临数百条行业信息，靠人工筛选既不现实也不可持续。

😰

传统方式

CEO每天花费 1-2 小时浏览新闻和行业报告，信息源分散在微信公众号、行业媒体、朋友圈，信息过载且无法持续追踪。关键趋势往往在爆发后才被注意到，错失决策窗口。

🧠

智闻方案

AI 驱动的行业情报系统，7×24 小时自动采集多源信息，AI 智能分析生成结构化周报/月报，识别风险与机会，主动推送决策简报。从「人找信息」变成「信息找人」。

01 📋 主题搜索 — 两级分类 + 自定义周期

这是智闻的起点。CEO 关注的行业不是单一的——他需要同时追踪「最新AI动态」「人形机器人产业」「量子计算商业化」等多个方向，每个方向内部还有细分。

两级结构：大类（如「AI科技前沿」）下可设小类（如「大模型」「Agent」「多模态」），形成树状分类体系。每个小类独立配置关键词、关联的新闻源和排除词，互不干扰。

自定义周期：报告周期不再是固定的「7天」，而是按自然日历精准切分——日报（今日）、周报（本周）、月报（本月）、季报、年报，每种周期独立为每个主题配置。每月1号只生月报不刷日报，保证每个报告都是语义完整的自然周期。

02 📚 归纳汇总 — 从碎片到结构

原始资讯是零散的——几十篇不同的文章、不同的角度、不同的信源。智闻在第一步先把这些碎片整理成有条理的归纳。

怎么做：将匹配到的文章按主题分类汇总，剔除重复和无关内容，提取关键信息。每篇文章保留标题、摘要、原文链接作为引用依据，生成结构化的文章快照列表。空周期时自动回溯7天查找文章，保证报告不出现空内容。

设计意图：CEO 不关心每篇新闻的原文细节，他需要的是一个高度浓缩的信息摘要——这个主题今天/本周发生了什么事，哪些是值得关注的。归纳汇总就是这个「减噪」的过程，把几十篇变成一个清晰的轮廓。

信息降噪去重摘要生成文章快照空周期回溯结构化整理

03 🔬 深度分析 — 客观真实 + 交叉验证

这是智闻区别于「摘要工具」的分水岭。普通工具只是陈列信息，智闻要理解信息背后的含义。

基本原则：分析必须客观真实，不能胡编乱造。AI 的输出必须有文章支撑，每条结论都要标注引用来源，让 CEO 能追溯到原始信息。

交叉验证机制：同一个事件从不同信源交叉比对——36氪报道的融资新闻 + 雷锋网的技术点评 + iThome的市场分析，多角度验证后得出的结论才有可信度。单源信息标记为「仅供参考」，多源验证的信息标记为「已交叉验证」。

安全阀设计：如果 AI 分析结果为空（所有字段无实质内容），报告不进展示、不推送，避免「空洞报告」浪费 CEO 时间。同时 AI 每次输出都需要有具体的文章引用，不输出无法溯源的空泛结论。

04 📊 SWOT 分析 — 风险预警 + 机会建议 + 解决方案

分析不是终点，决策才是。SWOT 四象限把零散信息转化为可行动的战略洞见。

SWOT 四象限：

S 优势 — 企业在当前趋势中的有利位置
W 劣势 — 企业面临的短板和挑战
O 机会 — 可把握的市场机会和增长点
T 威胁 — 外部风险因素和潜在危机

风险预警：识别出近期最值得关注的风险项（最多3条），每条附带风险等级（高/中/低）和对应文章链接。风险等级用红/黄/绿三色一目了然。

机会建议：从行业动态中提炼出可行动的商业机会（最多3条），每项附上建议和参考依据。

方法论支撑：SWOT + 风险/机会的框架本身就是一套完整的决策方法论，而非单纯的数据堆砌。每一份分析报告都是按这套方法论生产的标准化输出，CEO 接收到的不是信息本身，而是经过加工的决策素材。

SWOT四象限风险预警机会发现风险等级决策方法论可执行建议

05 📨 推送触达 — 图文报告直达飞书

报告写在系统里没人看 = 没有价值。智闻的最后一步是把分析成果推送到 CEO 日常使用的飞书上。

可视化报告：飞书 interactive 消息卡片，包含摘要 → SWOT 四象限 → 风险识别（🔴🟡🟢图标）→ 机会发现 → 源文章快照（前10篇带链接）。卡片头部颜色随风险等级变化，红/黄/绿一眼识别紧急程度。

自动 + 手动：报告生成后自动推送，同时支持 CEO 在系统内手动触发推送。OAuth API 和 Webhook 双通道容错，任一路径失败不影响另一路。

三段式输出：AI 生成的报告在展示端有三种形态——系统 Web 端查看（完整版带交互）、飞书消息卡片（精炼版直达聊天）、PDF 下载（存档版可打印/转发），满足不同场景需求。

🧠

长期记忆系统

这是智闻区别于一次性分析工具的最重要特征。CEO 的报告不是孤立的——今天的周报需要参考上周的趋势判断，这个月的月报应该知道上个月关注什么。但 AI 对话有上下文窗口限制，上一轮的深度分析在下一轮就「忘记」了。

🎯 记住什么
  • CEO 关注的行业偏好和信号偏好
  • 哪些关键词命中率高/低（反馈信号）
  • 历史报告中哪些分析被特别关注
  • 风险偏好的逐步校准（高/中/低阈值）
  • 长期的趋势变化和数据积累
📈 沉淀的价值
  • 3个月的报告连起来可做趋势追踪
  • 6个月可发现周期性规律
  • 12个月形成完整的行业年度洞察
  • 数据越久越完整，分析越来越精准
  • 「这个趋势我们3个月前就预警过」成为可能

⚡ 所有报告在数据库中永久保存（回收站软删除可恢复），支持跨期查询和趋势对比。
每一次推送都不是终点，而是长期洞察的一块拼图。

Capabilities

核心优势

普通的信息聚合工具 ≠ 智能情报系统。智闻的每一项能力都围绕「决策效率」设计。

📚

多主题并行追踪

支持创建任意数量的行业主题，每个主题独立配置关键词、新闻源和报告周期，互不干扰。可同时追踪「机器人出海」「AI最新动态」「新能源汽车」等行业。

📅

5级日历周期

日报/周报/月报/季报/年报，使用自然日历范围，自动错开重叠周期。每月1号只生月报不生成日报和周报，避免重复。

🎯

AI SWOT 分析

DeepSeek 驱动的结构化分析引擎，输出优势/劣势/机会/威胁四象限，附带风险清单和商业机会清单，每项都引用源文章支撑。

🔔

主动推送到飞书

报告生成后自动推送到企业飞书群，CEO 无需登录系统即可在聊天中阅读。卡片包含摘要、风险等级标识和源文章链接。

⏰

精细化自动调度

每个主题三个独立开关：数据采集、报告生成、飞书推送。可独立启停，APScheduler 实时生效，无需重启服务。

🔒

数据安全可靠

单机 SQLite 部署，无需外部数据库。每日自动全量备份，保留30天历史。Git 实时推送代码到 GitHub，双重保障。

⚡

一键手动刷新

支持手动触发全量刷新：采集最新 RSS → 关键词匹配 → 生成 AI 报告。修改关键词后可立即重跑，无需等待定时调度。

🔄

空周期自动回溯

当前周期无数据时自动回溯7天查找文章，保证报告不出现空内容。回溯数据在报告中标注「含回顾」标记。

🔐

多组织隔离

基于 org_id 的数据隔离，每个注册用户自动分配独立组织。不同企业的主题、报告、推送配置完全隔离。
