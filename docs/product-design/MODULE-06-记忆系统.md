# Module 6: 长期记忆系统
## 产品设计文档 · 补充模块

> **核心定位：** 数字员工的"大脑皮层"——让AI越用越懂CEO，越用越有价值

---

## 1. 为什么需要长期记忆

### 当前问题

```
传统AI助手（无记忆）：
  第1次对话：CEO说"我叫张总，是XX公司CEO"
  第50次对话：AI问"请问您贵姓？" ❌

传统报告（无累积）：
  今天报告："原材料价格上涨"
  一个月后报告："原材料价格持续上涨"
  三个月后报告："原材料价格走势" ← 没有关联分析！❌
```

### 长期记忆的价值

```
有记忆的数字员工：
  第1次：了解CEO是XX公司一把手，关注AI和新能源赛道
  第50次：知道张总偏好简洁结论、每周一早上阅读、关注竞品动态
  第100次：能说"这是您第3次问原材料问题，这次加上了一个新变量..."

报告（带累积）：
  今天：原材料价格追踪（首次建立档案）
  1个月后：价格走势分析（30个数据点）
  3个月后：周期性规律分析 + 预测模型（90+数据点）→ 真正有价值的洞察
  1年后：跨年度对比 + 战略建议（有历史深度）→ 不可替代的价值
```

## 3. 四大记忆模块

### 3.1 CEO画像记忆

**存储内容：**

| 维度 | 示例 | 更新频率 |
|------|------|---------|
| 基本信息 | 姓名、公司、职位、行业 | 首次 |
| 沟通偏好 | 喜欢简洁结论 / 详细数据 | 持续观察 |
| 阅读习惯 | 早上通勤阅读 / 晚间深度研究 | 持续观察 |
| 决策风格 | 数据驱动 / 风险厌恶 / 快速决策 | 持续观察 |
| 关注重点 | AI赛道、竞品动态、政策合规 | 持续观察 |

| 禁忌事项 | "不要给我发邮件" / "不喜欢太技术化" | 首次+更新 |

## 4. 记忆调用机制

### 4.1 记忆检索（Memory Retrieval）

```
Agent处理请求时：
  
  Step 1: 提取查询关键词
          "分析一下最近竞品X的融资动态"
  
  Step 2: 向量检索（Embedding）
          在记忆中搜索相关记忆
          - CEO画像（竞品关注度）
          - 企业档案（已有竞品数据）
          - 报告档案（历史追踪记录）
          - 会话历史（上次讨论内容）
  
  Step 3: 相关记忆排序
          相关度 > 0.7 → 作为上下文注入
  
  Step 4: 生成回复
          结合记忆 + 新信息 → 更懂CEO的回复
```

### 4.2 记忆更新（Memory Update）

```
触发时机：
  - CEO明确提供信息（"我是XX公司CEO"）→ 立即更新
  - AI观察到CEO行为（"张总总是早上看报告"）→ 持续学习
  - 报告生成完成 → 自动沉淀到报告档案
  - 重大事件发生 → 自动更新企业档案
  - 会话结束/重要结论 → 自动保存到会话记忆
```

### 4.3 记忆衰减 & 强化

```
重要记忆（权重高）：
  - CEO明确表达的偏好/禁忌
  - 企业核心数据（营收、战略）
  - 长期追踪报告（数据点）
  → 长期保留，定期强化

一般记忆（权重低）：
  - 日常对话内容
  - 普通问题/回答
  → 6个月后降权，1年后归档或删除

过期的记忆：
  - 具体数值（"Q1营收是X亿"）→ 更新为最新值
  - 已不适用的判断（"竞品威胁低"）→ 标记为历史
```

## 6. 技术实现

### 6.1 技术选型

| 组件 | 选型 | 用途 |
|------|------|------|
| 记忆存储 | PostgreSQL + pgvector | 结构化数据 + 向量检索 |
| 向量数据库 | pgvector（轻量）/ Milvus（生产） | 记忆语义检索 |
| 缓存层 | Redis | 热记忆快速访问 |
| 记忆更新 | DeepSeek Function Calling | 触发记忆读写 |
| 历史报告 | PostgreSQL JSONB | 非结构化报告存储 |

### 6.2 数据模型

```sql
-- CEO画像表
CREATE TABLE ceo_profiles (
  id SERIAL PRIMARY KEY,
  user_id INT REFERENCES users(id),
  profile JSONB, -- 偏好/习惯/禁忌
  embedding VECTOR(1536), -- 向量
  updated_at TIMESTAMP DEFAULT NOW()
);

-- 企业档案表
CREATE TABLE company_profiles (
  id SERIAL PRIMARY KEY,
  user_id INT REFERENCES users(id),
  company_data JSONB, -- 企业信息
  embedding VECTOR(1536),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- 报告追踪表
CREATE TABLE report_tracking (
  id SERIAL PRIMARY KEY,
  user_id INT REFERENCES users(id),
  topic VARCHAR(255), -- 追踪主题
  tracking_data JSONB, -- 时间序列数据
  insights JSONB, -- AI生成的洞察
  start_date DATE,
  updated_at TIMESTAMP DEFAULT NOW()
);

-- 会话记忆表
CREATE TABLE session_memories (
  id SERIAL PRIMARY KEY,
  user_id INT REFERENCES users(id),
  session_id VARCHAR(255),
  summary TEXT,
  key_points JSONB,
  action_items JSONB,
  created_at TIMESTAMP DEFAULT NOW()
);
```

## 7. 交互示例

### 示例：记忆驱动的报告

```
无记忆报告：
  "今日原材料价格上涨2%，建议关注。"

有记忆报告：
  "🔥 今日重点：原材料价格上涨2%（已连续上涨第4周）
   
   📊 追踪档案（张总，这个主题您已关注87天）：
   - 成本累计上涨：+15.3%
   - 对标历史：2019年类似周期，成本↑18%后，贵司毛利率压缩了3.2pp
   - 当前判断：已接近您上次的关注阈值（15%）
   
   💡 建议行动（基于您的决策风格）：
   1. 短期：启动期货锁价60%敞口（您偏好风险前置）
   2. 中期：评估供应商替代（成本优先）
   3. 报告给您的截止：本周五（您周一早上看报告的习惯）
   
   ⚠️ 上次您的反馈是'希望看到具体数字和历史对比'，
   本次已按此偏好调整。"
```

## 8. 价值总结

| 报告价值 | 单次快照 | 累积洞察 |
| 决策支持 | 碎片化 | 有历史深度 |
| 用户体验 | "AI不懂我" | "它真的懂" |
| 长期价值 | 工具 | 不可替代的助手 |
