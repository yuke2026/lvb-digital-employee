# 数字员工下一阶段迭代规划

> **目标：** 从「会说话的工具」升级为「能干活的企业员工」

**核心差距：** 当前系统只能被动回答问题，无法主动执行任务、集成企业系统、自动闭环工作流。

---

## 第一阶段：执行力建设（核心）

### 价值逻辑
```
对话 → 理解意图 → 拆解任务 → 调用工具 → 执行闭环 → 汇报结果
```

### Task 1: 任务执行引擎架构

**目标：** 构建 Agent 的「手和脚」—— 工具调用层

**架构：**
```
Agent Brain
    ↓ tool_call
Tool Executor Layer（工具执行层）
    ├── HTTP Tool（调用外部API）
    ├── Code Interpreter（执行Python/JS代码）
    ├── File Tool（读写文件、数据处理）
    ├── Database Tool（查询/写入企业数据库）
    └── Web Scraping Tool（抓取网页数据）
    ↓
Result Reporter（结果汇报层）
```

**实现方案：** 在 `backend/app/agents/` 下新增 `tools/` 目录，每个工具独立模块化。

---

### Task 2: 企业飞书集成

**目标：** Agent 能操作飞书，实现「飞书即入口」

**功能清单：**
- ✅ 读取飞书消息，Agent 被动响应群聊 @mentions
- ✅ 主动推送消息到指定群/用户
- ✅ 创建/查询飞书日程
- ✅ 读取飞书文档内容

**技术路径：**
```python
# 飞书开放平台 API
POST https://open.feishu.cn/open-apis/im/v1/messages
GET  https://open.feishu.cn/open-apis/calendar/v4/calendars
```

---

### Task 3: 企业数据库查询

**目标：** Agent 能读懂企业数据，回答数据驱动的问题

**场景示例：**
- 员工问："我们Q1营收多少？" → Agent 自动查数据库 → 自然语言回答
- 问："哪个客户最近30天没下单？" → Agent 写SQL查ERP → 回复

**实现方案：**
- 封装 SQL Executor Tool
- 预设常见查询模板（营收、客户、库存）
- 对接 MySQL/PostgreSQL

---

### Task 4: 主动推送能力

**目标：** Agent 不只是被问，还能主动提醒

**场景：**
- "每天早上9点汇报昨日销售数据"
- "客户下单后自动发确认消息"
- "库存低于阈值自动报警到群"

**技术方案：** 基于定时任务（cron）+ Agent 主动发消息

## 第二阶段：CEO行业专家助手（新增重点）

### 场景描述

**用户画像：** 公司CEO、企业高管

**核心价值：** 7×24小时行业情报哨兵，自主监控政治/经济/行业动态，自动汇总分析，push给CEO做决策参考

### 功能架构
```
┌─────────────────────────────────────────────────────┐
│                  CEO 行业专家助手                     │
├─────────────────────────────────────────────────────┤
│  数据采集层（主动抓取）                                │
│  ├── 宏观政治：新华社、人民网、政府政策文件             │
│  ├── 宏观经济：国家统计局、Wind、宏观数据              │
│  ├── 行业资讯：艾瑞、36Kr、行业报告                   │
│  ├── 竞争对手：新闻监控、财报分析                      │
│  └── 资本市场：股价、研报、PE/VC动态                   │
├─────────────────────────────────────────────────────┤
│  AI分析层（DeepSeek推理）                             │
│  ├── 信息筛选去重 + 重要性评分                        │
│  ├── 趋势判断 + 风险预警                              │
│  └── 结构化报告生成（摘要+结论+建议）                  │
├─────────────────────────────────────────────────────┤
│  推送层（主动push到CEO）                              │
│  ├── 早报：每天8:30 定时推送                          │
│  ├── 晚报：每天18:00 定时推送                         │
│  ├── 突发：重大事件立即push                           │
│  └── 定制：CEO随时提问，秒级响应                      │
└─────────────────────────────────────────────────────┘
```

### 三大产品形态

| 形态 | 触发方式 | 场景 |
|------|---------|------|
| **早晚报** | 定时自动推送 | 每天CEO上班前收到行业简报 |
| **即时分析** | CEO随时提问 | "帮我分析下这个政策对行业的影响" |
| **预警通知** | 关键词/条件触发 | "竞品融资、监管政策突变、行业数据异动" |

### Task 6: 情报采集工具集

**目标：** Agent 能自主抓取多源信息

**工具清单：**
- `WebSearchTool` — 搜索新闻、报告、政策文件
- `RSSMonitorTool` — 监控RSS/Atom信息源
- `APIScraperTool` — 抓取宏观数据（统计局API、Wind）
- `PDFReaderTool` — 读取PDF报告（券商研报、行业白皮书）

**数据源示例：**
```
政治：新华/人民网政策频道、发改委/商务部官网
经济：国家统计局月度数据、Wind宏观数据
行业：艾瑞咨询、36Kr、动脉网、行业垂直媒体
竞品：企业官网新闻、财报、公告
```

### Task 7: AI分析引擎

**目标：** 把碎片信息整合成结构化洞察

**分析维度：**
- **摘要生成**：50字核心要点
- **趋势判断**：环比/同比变化、趋势预判
- **影响评估**：对公司的正面/负面影响（1-5分）
- **建议行动**：建议CEO做什么（具体可执行）

**输出格式：**
```
📊 【行业日报】2025-05-22

🔥 今日重点
1. [政策] 工信部发布AI行业新规（影响评分：4/5）
   → 简评：监管趋严，头部玩家受益...
2. [资本] 竞品X公司完成B轮融资3亿（影响评分：3/5）
   → 简评：...

📈 数据速览
- 行业PMI：52.1（↑0.3）
- 原材料价格：+2.1%（↑）

💡 建议行动
1. 关注政策合规，检查现有产品...
2. 加速融资节奏，警惕资金压力...

📅 近期关注
- 5月25日 行业峰会
- 6月1日 竞品发布会
```

### Task 8: 定时推送系统

**目标：** 早上8:30 / 晚6:00自动推送到CEO飞书

**技术方案：**
```python
# APScheduler 定时任务
from apscheduler.schedulers.asyncio import AsyncIOScheduler

scheduler = AsyncIOScheduler()
scheduler.add_job(send_morning_brief, 'cron', hour=8, minute=30)  # 早报
scheduler.add_job(send_evening_brief, 'cron', hour=18, minute=0)  # 晚报
```

## 第三阶段：多Agent协作（进阶）

### 价值逻辑
```
主Agent（调度者）
    ↓ 分解任务
子Agent A（查数据） + 子Agent B（写文案） + 子Agent C（发邮件）
    ↓ 结果汇总
主Agent → 整合输出 → 交付
```

### Task 5: 主从Agent架构

**实现：** 主Agent负责任务分解和结果整合，从Agent并行执行子任务。
